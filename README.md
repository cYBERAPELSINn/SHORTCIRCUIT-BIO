# SHORTCIRCUIT Bio

Готовая статическая страница для GitHub Pages.

## Важно

Эта версия использует `Material Web` через CDN-модули.

Из-за этого локально ее лучше открывать не двойным кликом по `index.html`, а через маленький HTTP-сервер.
На GitHub Pages все будет работать нормально.

## Что внутри

- `index.html` - основная страница
- `styles.css` - стили
- `script.js` - вкладки и копирование ников
- `.nojekyll` - отключает лишнюю обработку GitHub Pages

## Как выложить на GitHub Pages

1. Создай новый репозиторий на GitHub.
2. Нажми `Add file` -> `Upload files`.
3. Загрузи в корень репозитория файлы:
   - `index.html`
   - `styles.css`
   - `script.js`
   - `.nojekyll`
4. Нажми `Commit changes`.
5. Открой `Settings` репозитория.
6. Перейди в раздел `Pages`.
7. В блоке `Build and deployment` выбери:
   - `Source`: `Deploy from a branch`
   - `Branch`: `main`
   - `Folder`: `/ (root)`
8. Нажми `Save`.
9. Подожди 1-3 минуты.
10. Ссылка на сайт появится в том же разделе `Pages`.

## Как посмотреть локально

Если просто открыть `index.html` файлом, некоторые браузеры могут некорректно загрузить `Material Web` компоненты.

Сделай так:

1. Открой PowerShell в папке с сайтом.
2. Выполни:

```powershell
python -m http.server 8123
```

3. Открой в браузере:

`http://127.0.0.1:8123/`

4. Чтобы остановить сервер, нажми `Ctrl + C`.

## Если сайт не открывается

- Проверь, что файл называется именно `index.html`
- Проверь, что он лежит в корне репозитория, не внутри папки
- Проверь, что выбран `main` и папка `/ (root)`
- После включения Pages подожди пару минут и обнови страницу
- Если был старый неудачный деплой, попробуй заново нажать `Save` в `Pages`
- Если локально страница выглядит сломанной, проверь, что ты открыл ее через `http://127.0.0.1:8123/`, а не через `file:///...`

## Если хочешь свой красивый адрес

Репозиторий можно назвать:

- `shortcircuit-bio`
- `shortcirquit-site`
- `cyberapelsin`

Тогда адрес будет примерно такой:

`https://твой-логин.github.io/shortcircuit-bio/`

## Быстрый способ через GitHub Desktop

1. Создай пустой репозиторий на GitHub.
2. Открой GitHub Desktop.
3. Клонируй этот репозиторий на компьютер.
4. Скопируй туда файлы сайта.
5. Сделай `Commit to main`.
6. Нажми `Push origin`.
7. Включи GitHub Pages в `Settings` -> `Pages`.

## Что можно менять самому

- Текст и блоки: `index.html`
- Цвета и внешний вид: `styles.css`
- Поведение вкладок: `script.js`
